## Content Standards

### Titles (under 60 chars)
Format: `[State] Egg Donor Pay [Year]: Earn $X,XXX–$X,XXX`
Or: `[State] Surrogate Pay [Year]: Earn $X,XXX–$X,XXX`
Example: `Illinois Egg Donor Pay 2026: Earn $8,000–$12,000`
Never: append site name (EggSurrogatePay.com), lead with "How much do egg donors make...", duplicate across states

### Meta Descriptions (140–155 chars)
Lead with earnings amount. Include state, role, and one secondary detail (requirements or timeline).
Template: `Earn $X,XXX–$X,XXX as an egg donor in [State]. See requirements, timeline, and how to apply for 2026 programs.`
Or: `Earn $X,XXX–$X,XXX as a surrogate in [State]. See base pay, allowances, and requirements for 2026 placements.`
Never: "How much do X make in Y?", site name, duplicate across pages.

### Internal Linking — Every New Page Must Have
1. A link from its **hub page** — egg donor guides from `/egg-donor-pay.html`; surrogate guides from `/surrogate-pay.html`
2. An entry in **sitemap.xml**
3. At least one **contextual in-body link** from a related page (adjacent state guide or hub page) with keyword anchor text (e.g. "egg donor pay in Illinois")
4. The generator template already links back to the hub page — verify this is in place when modifying the template

### Page Types and Hub Mapping
| New page type | Must be linked from |
|---|---|
| Egg donor state guide (via `generate_state_pages.py`) | `/egg-donor-pay.html` hub + sitemap |
| Surrogate state guide (via `generate_state_pages.py`) | `/surrogate-pay.html` hub + sitemap |
| Hand-written content page | Homepage nav + relevant hub page |

### Generator Note
When modifying `generate_state_pages.py` templates: update both the child page template (breadcrumb/hub link) and the hub page template (child page links) in the same commit.
